Reacher AI

How the system works

A managed AI calling service — inbound and outbound. Contacts enter through uploads, web forms, or scraping; one engine qualifies, calls, books meetings, sends reminders, and throttles capacity. Every node is connected — one continuous, self-regulating system.

0 Contact lifecycle (state machine)

stateDiagram-v2 [*] --> new: entered (upload / form / scrape) new --> enriched: researched (scraping pipe only) new --> scrubbed: clean list — skip enrichment enriched --> scrubbed: passed opt-out gate enriched --> disqualified: failed ICP or on DNC scrubbed --> calling: picked up by daily dialer calling --> booked: meeting set ✓ calling --> not_interested: declined calling --> no_answer: missed / busy / voicemail no_answer --> calling: retry (≤3, +3 days, rolls past weekends) no_answer --> [*]: exhausted → follow-up list booked --> [*] not_interested --> [*] disqualified --> [*]

The state column IS the queue — a DB query in priority order drives the daily run. Uploaded contacts skip enriched. Inbound calls land directly as booked.

Example: 20 retries due + 100 fresh → dial 16 retries + 24 fresh = 40 total. 4 retries roll to tomorrow.
